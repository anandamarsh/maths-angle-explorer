## Data types (`sessionLog.ts`)

### `QuestionAttempt`

```ts
export interface QuestionAttempt {
  questionNumber: number;
  prompt: string;           // question text shown to player
  level: 1 | 2 | 3;
  correctAnswer: number;    // correct angle in degrees
  childAnswer: number | null; // typed angle (null if aim-only drag shot)
  isCorrect: boolean;
  timestamp: number;
  timeTakenMs: number;      // ms from startQuestionTimer() to logAttempt()
  gamePhase: "normal" | "monster" | "platinum";
  // Level 2 data for PDF diagram:
  sectorArcs?: AngleSector[];
  dividerAngles?: number[];
  totalContext?: 90 | 180 | 360;
  startAngleDeg?: number;
  setKind?: "COMPLEMENTARY" | "SUPPLEMENTARY" | "COMPLETE";
}
```

### `SessionSummary`

```ts
export interface SessionSummary {
  playerName: string;
  level: 1 | 2 | 3;
  date: string;           // ISO timestamp
  startTime: number;      // Date.now() at session start
  endTime: number;        // Date.now() at buildSummary()
  totalQuestions: number;
  correctCount: number;
  accuracy: number;       // 0-100, rounded
  normalEggs: number;     // stars earned in normal round
  monsterEggs: number;    // stars earned in monster round
  levelCompleted: boolean;
  monsterRoundCompleted: boolean;
  attempts: QuestionAttempt[];
}
```

## PDF generation (`generatePdf.ts`)

```ts
export async function generateSessionPdf(summary: SessionSummary): Promise<Blob>
```

Returns an A4 portrait PDF blob using jsPDF 4.

### PDF sections (top to bottom)

1. **Header banner** (28mm tall, `#f1f5f9` bg)
   - App icon (20×20mm, loaded from `/favicon.svg` → fallback `/icon-512.png`)
   - Title: "Angle Explorer" (17pt bold)
   - Subtitle: "Level N Session Report" (9pt bold)
   - Date + session time range (7.5pt, muted)

2. **Game description**
   - "About This Game" heading (9pt bold)
   - Objective line: depends on level (angle classification, missing angles, etc.)

3. **Score boxes** (3 equal boxes, 18mm tall)
   - Score: blue (`#1d4ed8`)
   - Accuracy: green ≥80%, amber ≥50%, red <50%
   - Time: purple (`#a855f7`)

4. **Star row** — star icons: correct=yellow `#facc15`, wrong=red `#ef4444`

5. **Question cards** (one per attempt)
   - Header: coloured left stripe, Q number, CORRECT/WRONG label, time taken
   - Body: angle diagram (left, 70×42mm) + question text & answers (right)

6. **Encouragement section** (lavender `#ede9fe`)
   - ≥90% → "Outstanding work! 🌟"
   - ≥70% → "Great effort! Keep it up! 🎉"
   - ≥50% → "Good try! Practice makes perfect! 💪"
   - <50% → "Keep going — you're learning! 🌱"
   - Tip line if any wrong answers

7. **Footer** — "Generated by SeeMaths" + `www.seemaths.com`

### Angle diagram (per question card)

Replaces the ripple-position diagram from the template:

**Level 1 questions:** Renders a protractor-style diagram:
- Origin point at diagram left
- Reference ray (0°, pointing right)
- Answer ray at `correctAnswer` degrees
- Arc showing the angle span
- Label: `${correctAnswer}°`

**Level 2 questions:** Renders a sector pie diagram:
- Sector arcs drawn from `startAngleDeg` for each sector in `sectorArcs`
- Known sectors: light fill + angle label
- Missing sector: highlighted differently + `?` label + answer below
- Total context label (e.g. "SUPPLEMENTARY 180°")

## Email API (`api/send-report.ts`)

**Endpoint:** `POST /api/send-report`
**Body size limit:** 10MB (for base64 PDF)

**Required env vars:**
- `RESEND_API_KEY` — Resend API key
- `EMAIL_FROM` — verified sender address

**Validation:**
- Email format: `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`
- `pdfBase64` must be non-empty

**Request body:**
```ts
{
  email, pdfBase64,
  playerName, correctCount, totalQuestions, accuracy,
  gameName, senderName, siteUrl,
  sessionTime, sessionDate, durationText,
  stageLabel, curriculumCode, curriculumDescription,
  curriculumUrl, curriculumIndexUrl,
  reportFileName,
  emailSubject, emailGreeting, emailBody, emailBodyHtml,
  emailCurriculum, emailCurriculumHtml, emailRegards,
}
```

The endpoint builds the final email HTML. The frontend may send pre-localized plain-text
and/or HTML strings; when localized HTML is provided, the endpoint should preserve the
same Ripple Touch-style formatting:
- body sentence with inline `SeeMaths` link
- bold game name, time, date, duration, score, and accuracy
- curriculum sentence with bold linked stage and bold linked curriculum text
- `{playerName} played ...` when a name exists, otherwise `A player played ...`

Both `api/send-report.ts` and the local Vite `/api/send-report` middleware must stay in
sync so local preview matches Vercel.
